文档流

  1. 流动方向

    • inline 元素从左到右，到达最右边才会换行。

    • block 元素从上到下，没一个元素都占独立的一行，新元素会另起一行。

    • inline-block 元素从左到右，到达最右边才会换行，但不会强行拆分元素。

      • 新的 HTML5 里可以不分内联元素和块级元素，具体情况由默认样式决定，可以设定更改。

      • inline-block 元素不会有外边距合并现象。

      • 非特定情况下永远不要写width: 100%。

  2. 宽度

    • inline 宽度为内部 inline 元素的和自适应，不能用 width 属性进行设置。
    • block 默认自动计算宽度，可以用 width 属性进行设置。
    • inline-block 结合前两者特点，可以设置宽度。

  3. 高度

    • inline 高度由 line-height 间接决定，跟 height 无关。
    • block 高度由内部文档流元素决定，可以用 height 值进行设置。
    • inline-block 与 block 类似，可以设置高度。

  4. overflow

    当内容的宽度或高度大于容器，则会溢出，可以用overflow 属性来设置溢出内容的显示方式。

    • overflow: auto 灵活设置，当内容溢出时自动显示滚动条，否则滚动条界面消失。
    • overflow: scroll 永远显示滚动条。
    • overflow: hidden 隐藏溢出部分的内容。
    • overflow: visible 直接在页面上显示溢出内容。
    • overflow 可以分为 overflow-x 和 overflow-y

  5. 脱离文档流

    可以使用 float 属性 right/left 和 position 属性 absolute/fixed 来使某个元素脱离文档流，这样父级元素或容器计算尺寸时会忽略该元素。

盒模型

CSS盒模型本质上是一个盒子，封装周围的HTML元素，它包括：边距margin，边框border，内填充 padding，和实际内容 content。

  • Margin(外边距) - 清除边框外的区域，外边距是透明的。

  • Border(边框) - 围绕在内边距和内容外的边框。

  • Padding(内边距) - 清除内容周围的区域，内边距是透明的。

  • Content(内容) - 盒子的内容，显示文本和图像。

盒模型在实际使用中根据计算方式的不同又分为标准盒模型（content-box）和怪异盒模型（border-box）。

  • 标准盒模型

    盒模型高度和宽度 = 内容高度和宽度

  • 怪异盒模型

    盒模型高度和宽度 = 内容高度和宽度 + padding + border

    一般来说 border-box 用的更多一点

margin 合并

块级元素的顶部外边距和底部外边距有时被组合(折叠)为单个外边距，其大小是组合到其中的最大外边距，这种行为称为外边距合并（只合并顶部和底部外边距）。

  • 相邻兄弟元素合并

    阻止兄弟元素合并：添加 inline-block 属性。

  • 父子元素合并

    给父级元素添加 border/padding。

    给父级元素添加 display:flex。

    给父级元素添加 overflow: hidden。

浏览器渲染过程

  1. 根据 HTML 构建 HTML 树。（DOM）

  2. 根据 CSS 构建 CSS 树。（CSSOM）

  3. 将两棵树合并成一颗渲染树。（render tree）

  4. Layout 布局。（文档流、盒模型、计算大小和位置）

  5. Paint 绘制。（把边框颜色、文字眼、阴影等画出来）

  6. Compose 合成。（根据层叠关系展示画面）

如何更新样式

  • 一般使用 JS 来更新样式

    • 比如 div.style.background='red'
    • 比如 div.style.display='none'
    • 比如 div.classList.add='red'
    • 比如 div.remove() 可以直接删掉节点

  • 三种更新方式

    • 第一种，全走：

      比如 div.remove() 触发当前元素消失，其他所有元素 relayout。

    • 第二种，跳过 layout：

      比如 改变背景颜色，直接 repaint + composite。

    • 第三种，跳过 layout 和 paint

      比如 改变 transform，只需要 composite。

    查询具体触发流程：https://csstriggers.com/

  • CSS 动画优化

    1. JS 优化

      使用使用 requestAnimationFrame 代替 setInterval 或者 setTimeout。

    2. CSS 优化

      使用 will-change 或 translate

      [详细参考链接][https://developers.google.com/web/fundamentals/performance/rendering/stick-to-compositor-only-properties-and-manage-layer-count]

transform

  • 四个常用功能

    • 位移：translate

      • 常用写法

        1. translate() = translate(<length-percentage> , <length-percentage>?)

        2. translateX() = translateX(<length-percentage>

        3. translateY() = translateY(<length-percentage>

        4. translateZ() = translateZ(<length> 且父容器有 perspective属性。

        5. translate3d(x.y.z)

          translate(-50%, -50%)可做绝对定位元素的居中

    • 缩放：scale

      • 常用写法
        1. scale()=scale(<number> , <number>?)
        2. scaleX()=scaleX(<number>)
        3. scaleY()=scaleY(<number>)

    • 旋转：rotate

      • 常用写法

        1. rotate([<angle>|<zero>])

        2. rotateX([<angle>|<zero>])

        3. rotateY([<angle>|<zero>])

        4. rotateZ([<angle>|<zero>])

          一般用于 360 度旋转制作 loading

          [rotate MDN][https://developer.mozilla.org/zh-CN/docs/Web/CSS/transform-function/rotate()]

    • 倾斜：skew

      • 常用写法

        1. skewX([<angle>|<zero>])

        2. skewY([<angle>|<zero>])

        3. skew([<angle>|<zero>] , [<angle>|<zero>]?)

          [skew MDN][https://developer.mozilla.org/zh-CN/docs/Web/CSS/transform-function/skew()]

  • transform 多重效果

    • 组合使用
      • transfome:scale(0.5) translate(-100%,-100%)
      • transform:none 取消所有

transition

  • 作用：补充中间帧

  • 语法

    • transition: 属性名 时长 过渡方式 延迟

      transition: left 200ms linear

    • 可以用逗号分个两个不同属性

      transition: left 200ms, top 400ms

    • 可以用 all 代表所有属性

      transition:all 200ms

    • 过渡方式有: linear|ease|ease-in|ease-out|ease-in-out|cubic-bezier|step-start|step-end|steps

      [具体解释][https://developer.mozilla.org/zh-CN/docs/Web/CSS/timing-function]

  • 注意事项

    • 并不是所有属性都都能过渡
      1. display:none => 没办法使用动画过渡
      2. 一般改成visibility:hidden => visible

  • 过渡必须要有起始

    一般只有一次动画，或者两次，比如hover和非hover状态的过渡

  • 除了起始，含有中间点的过渡

    • 使用两次 transform
      • .a ===transform===> .b
      • .b ===transform===> .c
      • 用 setTimeout 或者监听 transitionend 事件监测中间点

animation

  • 声明关键帧@keyframes

    • 标准写法

      1. From to
      @keyframes slidein {
  from {
    transform: translateX(0%); 
  }
    
  to {
    transform: translateX(100%);
  }
}
      1. 百分数
      @keyframes identifier {
  0% { top: 0; left: 0; }
  30% { top: 50px; }
  68%, 72% { left: 50px; }
  100% { top: 100px; left: 100%; }
}

  • animation

    缩写语法

    animation:时长|过渡方式|延迟|次数|方向|填充模式|是否暂停|董动画名;

    • 时长：1s 或者 1000ms
    • 过渡方式：跟 transition 取值一样，入 linear
    • 次数 ：3 或者 2.4 或者 infinite
    • 方向：revere|alternate|alternate-reverse
    • 填充模式：none|forwars|backwards|both
    • 是否暂停：paused|running
    • 以上所有的属性都有对应的单独属性

    [Animation MDN][https://developer.mozilla.org/zh-cn/docs/web/css/animation]
